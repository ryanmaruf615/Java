# âš™ï¸ Config Schema (Conceptual)

This is the **structure** your external config file should support.
You can later decide format: `.json`, `.yaml`, or `.properties`.

---

### **Top-level fields**

* `scanFolder` â†’ which folder to organize.
* `categories` â†’ mapping of extensions to folder names.
* `defaultCategory` â†’ fallback if no mapping found.
* `conflictStrategy` â†’ how to handle duplicates (`rename`, `overwrite`, `skip`).
* `excludePatterns` â†’ list of regex/wildcards to skip.
* `hiddenFiles` â†’ whether to skip hidden/system files (`true`/`false`).
* `dryRun` â†’ default mode (`true`/`false`).
* `watchMode` â†’ run continuously? (`true`/`false`).
* `logFile` â†’ optional log path.
* `maxFileSizeMB` â†’ skip files above size limit (optional).

---

### **Example (YAML style, easier to read)**

```yaml
scanFolder: "C:/Users/Maruf/Downloads"

categories:
  pdf: "PDFs"
  jpg: "Images"
  jpeg: "Images"
  png: "Images"
  docx: "Documents"
  doc: "Documents"
  zip: "Archives"
  rar: "Archives"
  mp4: "Videos"
  mp3: "Music"

defaultCategory: "Others"

conflictStrategy: "rename"   # options: rename, overwrite, skip
excludePatterns:
  - ".*\\.tmp"       # skip temp files
  - ".*~"            # skip backup files

hiddenFiles: false
dryRun: true
watchMode: false
logFile: "organizer.log"
maxFileSizeMB: 500
```
